# Windows Study Radar Locking Agent 🔒

맥북에서의 공부 진행 상황(Firebase 실시간 데이터베이스)을 주기적으로 확인하여, 목표 학습 시간을 채우지 않았거나 오프라인 우회를 시도하는 경우 Windows PC 사용을 강제로 잠금 차단하는 백그라운드 에이전트입니다.

## ✨ 핵심 기능
1. **Windows 화면 강제 잠금 루프 🔒**
   - 2초 주기로 클라우드에서 공부 달성률을 체크합니다. 
   - 목표 미달 시 `ctypes.windll.user32.LockWorkStation()` API를 호출하여 즉각 컴퓨터를 시스템 잠금 화면으로 날려버립니다. 잠금을 해제하더라도 다시 즉시 잠금 상태가 됩니다.
2. **이중 프로세스 감시 부활 장치 (Watchdog Double-Daemon) 🛡️**
   - 찰나의 잠금 해제 타이밍에 작업 관리자(`Task Manager`)를 켜서 에이전트를 강제 종료하는 꼼수를 원천 차단합니다.
   - 실행 시 백그라운드에서 감시 프로세스(Watchdog)가 함께 가동되어, 메인 에이전트 프로세스가 죽는 즉시 1초 이내에 새 창으로 부활시킵니다.
   - 감시 프로세스 역시 메인 루프에서 생존 여부를 계속 폴링하며, 감시 프로세스가 죽을 경우 메인 프로세스가 즉각 재기동시킵니다.
3. **인터넷 연결 끊기 우회 차단 📡**
   - 공부하기 싫어서 윈도우 PC의 인터넷선(LAN)을 뽑거나 와이파이를 꺼버려 클라우드 동기화를 우회하려는 꼼수를 감지합니다.
   - 네트워크 연결 끊김이 3회(6초) 이상 지속되면 보안 조치로 화면을 강제로 잠가버립니다.
4. **부팅 시 자동 시작 등록 (Startup Autostart) ⚙️**
   - 윈도우 PC가 켜지자마자 사용자가 게임을 시작하기 전에 차단 데몬이 가동될 수 있도록 윈도우 레지스트리(`HKEY_CURRENT_USER\...\Run`)에 실행 파일 경로를 자동 등록합니다.

## 🛠️ Windows PC 설치 및 세팅 방법

집에 복귀하셔서 Windows PC에서 아래 순서대로 세팅을 완료해 주세요.

### 1. 패키지 설치
윈도우 터미널(CMD 또는 PowerShell)을 열고 의존성 패키지를 설치합니다.
```bash
pip install -r requirements.txt
```

### 2. Firebase 데이터베이스 주소 확인
- 이미 `/windows_agent/config.json` 파일에 사용 중이신 맥북과 동일한 Firebase 주소가 자동으로 보관 및 세팅되어 있습니다. 
- 혹시 DB가 바뀌는 경우 `config.json`의 주소만 수정해 주시면 됩니다.

### 3. 무설치 단일 실행 파일(.exe)로 빌드하기 (강력 추천)
사용자가 임의로 파이썬 코드를 읽거나 수정하지 못하게 하고, 검은색 콘솔 창 없이 완벽한 **백그라운드 백신 서비스처럼 가동**시키려면 `.exe` 실행 파일로 패키징해 두는 것이 좋습니다.

1. 빌드 패키지(`pyinstaller`)를 설치합니다:
   ```bash
   pip install pyinstaller
   ```
2. 콘솔 창이 없는 백그라운드 무설치 단일 `.exe` 파일로 컴파일합니다:
   ```bash
   pyinstaller --noconsole --onefile agent.py
   ```
3. 빌드가 완료되면 `dist/` 폴더 내에 `agent.exe` 파일이 생성됩니다.
4. 이 `agent.exe` 파일과 `config.json` 파일을 **동일한 폴더**에 두고 `agent.exe`를 처음 한 번 실행해 주면 자동으로 부팅 시마다 백그라운드에서 완벽하게 감시 및 잠금 루프가 작동하게 됩니다.

## 🚫 프로그램 비활성화 및 제어권 반환 방법
오늘 공부 목표(예: 3시간, 5시간 등)를 달성 완료하면 윈도우 PC는 즉시 정상 해제되며 자유롭게 게임을 즐기실 수 있습니다.

만약 공부 시간 측정 이외에 비상시에 강제로 감시 프로그램을 정지시키고 싶다면:
- **맥북 트레이**에서 타이머 앱의 목표 시간을 일시적으로 `0`분으로 조정해 Firebase에 업로드되도록 하면 윈도우 PC의 잠금 루프가 즉각 정상 해제됩니다.
- 또는 개발자 본인의 수동 접근 권한을 사용해 Firebase DB 내의 `date` 값을 임의로 가공하거나 데이터 구조를 임시 변경하면 감시가 잠시 풀립니다.
